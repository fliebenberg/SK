# Explicit Approval

When creating an `implementation_plan.md` in PLANNING mode, you MUST wait for the user to explicitly say "go ahead", "implement this", "looks good", or similar affirmative phrases before you switch to EXECUTION mode and begin modifying code. 

If the user does not provide explicit approval after you present a plan, or if you are unsure if their response constitutes approval, you must ask for clarification and wait for their explicit go-ahead. Do not auto-proceed.
